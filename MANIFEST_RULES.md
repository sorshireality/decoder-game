# Manifest Rules

## Обязательные правила

1. Перед началом любой новой задачи обязательно ознакомиться с файлами в порядке чтения из `/Users/oleksandrtiutiunnik/Documents/pet-projects/decoder-game/MANIFEST.MD`.
2. Любые изменения структуры приложения, экранов, состояния, логики уровней или геймплея должны быть отражены в соответствующем manifest-файле в этом же изменении.
3. Приоритет проекта: максимально легковесная и шустрая реализация (минимум зависимостей, быстрый рендер, простой стек).
4. В рамках этого проекта перед выполнением задачи/ответом указывать оценку неопределенности требований в диапазоне `0..1`.
5. Если оценка неопределенности `>= 0.3`, перед реализацией обязательно задать уточняющие вопросы.

## Обновление манифестов

Обновлять manifest-файлы при изменениях:
- структуры экранов и UI-потока
- логики уровней, стадий, сложности, попыток
- правил проверки результата
- способностей/бонусов
- i18n и текстов
- хранения данных / PWA / сетевого поведения
- добавления PvP, комнат, онлайна, backend-интеграции

## Стандарты качества

- Сохранять совместимость с single-player, если задача явно не меняет это.
- Не добавлять тяжелые зависимости без необходимости.
- Для PvP и рейтинга приоритетны строгие политики доступа (RLS) и предсказуемый state-flow.
