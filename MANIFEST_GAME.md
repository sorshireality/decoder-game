# Manifest Game

## Языки

- Поддержка: `en`, `uk`.

## Core gameplay

- Угадывание цветовой последовательности по позициям.
- Основной фидбек: `correct / total` (совпадение по позиции).
- Цвета: `red`, `orange`, `yellow`, `green`, `blue`, `purple`.

## Стадии и механики

- Всего уровней: `41`
- Стадий: `6`

1. `Stage 1: Classic` (1-10)
- Базовый режим, без лишнего цвета.

2. `Stage 2: Decoy` (11-15)
- В палитре +1 лишний цвет.

3. `Stage 3: Repeats` (16-23)
- Повторы цветов в секрете разрешены.

4. `Stage 4: Fog` (24-29)
- В истории видно только последние 3 попытки.

5. `Stage 5: Freeze` (30-35)
- Со 2-го хода после каждого confirm случайный слот заморожен на следующий ход.
- Замороженный слот нельзя менять.
- Ability игнорирует замороженный слот.
- Замороженный слот сохраняет предыдущее значение, чтобы confirm оставался доступным.

6. `Stage 6: Chain` (36-41)
- Один уровень состоит из серии мини-раундов.
- Каждый следующий раунд: меньше попыток, больше заранее открытых позиций.
- Проигрыш одного мини-раунда = проигрыш уровня.

## LEVELS v2

- Источник правды: `LEVELS_V2`.
- Поля: `feature`, `featureConfig`, `mode`, `scoring`, `stageLevel`, `globalLevel`.
- Legacy `LEVELS` сохранен как flattened-view (`lvl.game`).

## Ability

- Unlock после уровня 6.
- Один use на уровень.
- Открывает правильный цвет случайной неоткрытой позиции.

## Daily (MVP)

- Один общий челлендж в день (`UTC`, детерминированный seed).
- Одна зачетная попытка в день.
- Награда: `15 + 5 * attemptsLeft`.
- Состояние daily хранится локально в `decoder_daily_state`.

## Очки

### Main Game

- Только за `first clear` уровня (защита от фарма перепрохождением).
- Формула: `stageBasePoints * levelCoefficient * featureMultiplier`.

Feature multipliers:
- `none`: `1.00`
- `repeats`: `1.05`
- `fog`: `1.15`
- `freeze`: `1.20`
- `chain`: `1.30`
- `double_code`: `1.35` (зарезервировано)

### PvP

- Победа: `+20`
- Поражение: `+5`

## PvP (MVP)

- Лобби: `Find Match` / `Create`.
- Matchmaking: `Any room` или `By settings`.
- Автопоиск + fallback авто-создания комнаты.
- Room flow: `waiting_for_opponent` -> `ready` -> матч.
- Экран `VS` перед стартом.
- `Rematch` в той же комнате.
- Пока без полной синхронизации попыток/истории соперника в поле.
